Range and Precision:
Number: The Number data type represents double-precision 64-bit floating-point numbers (IEEE 754 standard).
It can accurately represent integers between -2^53 and 2^53, and fractional numbers with up to 15
significant digits. However, beyond this range, precision may be lost due to limitations in the floating-point representation.

BigInt: The BigInt data type was introduced in ECMAScript 2020 (ES11) to handle arbitrarily large integers.
It can represent integers with arbitrary precision, limited only by available memory. BigInt literals are created
by appending n to the end of an integer literal or by calling the BigInt() constructor.

Usage and Compatibility:
Number: The Number data type is widely supported in JavaScript and is suitable for most numeric operations,
including arithmetic calculations, comparisons, and bitwise operations. It's the default choice for representing numbers in JavaScript.

BigInt: The BigInt data type is relatively new and has limited support in older browsers and environments.
It's mainly used for scenarios where precise arithmetic on very large integers is required, such as cryptography,
numerical analysis, or dealing with numbers beyond the safe range of Number.

Operations and Compatibility:
Number: Number supports arithmetic operations, comparisons, and bitwise operations. It's compatible with most mathematical functions and operators in JavaScript.

BigInt: BigInt supports basic arithmetic operations (addition, subtraction, multiplication, division, and modulo),
but some operations that are supported for Number are not supported for BigInt. For example, bitwise operations
are not supported for BigInt, and you cannot mix BigInt and Number directly in operations.
